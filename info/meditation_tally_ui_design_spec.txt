content = """Meditation Tally App - UI Design Specification
=============================================

Document Purpose
----------------
This document defines the visual design direction, screen structure, interaction behavior,
and Jetpack Compose component breakdown for the approved Meditation Tally home screen design.

Project
-------
Meditation Tally

Design Version
--------------
Home Screen V1

Design Direction Summary
------------------------
The approved UI direction is a calm, soft, minimal meditation counter with:
- a light mint / sage visual theme
- a large central count circle
- a subtle progress arc around the circle
- no redundant +1 button inside the circle
- the entire main circle acting as the primary tap target
- a small goal label such as "20 / 108" for progress clarity
- softer secondary actions for Undo, Reset, and History

Primary Design Goals
--------------------
- Make the count the visual focus
- Make the tap interaction feel obvious and satisfying
- Keep the interface calm and uncluttered
- Support one-hand use
- Ensure the main interaction is faster than any secondary action
- Make daily goal progress understandable at a glance

Approved UI Changes
-------------------
Relative to the earlier concept, the following design decisions are now approved:

1. Remove the redundant +1 button
   - The main central circle becomes the primary tap button.
   - Tapping the circle increments the tally by 1.

2. Make the tap button more prominent
   - The circle must clearly read as an interactive surface.
   - Use stronger shadow, subtle glow, or slight elevation.
   - Add pressed-state scale animation and optional haptic feedback.

3. Add goal label
   - Display current count versus target as text such as "20 / 108".
   - Place it below the circle or just above the action row.
   - Keep the label subtle but easy to read.

4. Keep Undo visually softer than the main tap target
   - Secondary buttons must not compete with the circle.
   - Use lighter emphasis, smaller scale, and reduced contrast.

5. Use confirmation dialog for Reset
   - Reset must never happen instantly.
   - Show a confirmation dialog before resetting today's count.

6. Add pressed animation and haptic feedback
   - The tap circle should provide tactile and visual feedback.
   - Use subtle scale-down and release animation.
   - Trigger haptic feedback on successful increment when enabled.

Design Principles
-----------------
- Minimal, not empty
- Soft and calming
- Clear hierarchy
- Accessible touch targets
- Strong single primary action
- Secondary controls remain available but visually quiet

Visual Style Specification
--------------------------
Theme
- Meditation-focused
- Soft mint / sage palette
- Gentle glow and blur feeling
- Rounded geometry
- Airy spacing
- Minimal typography

Color Direction
---------------
Background
- Very light mint gradient
- Slight noise or soft vignette effect is optional

Primary Circle
- Soft green gradient fill
- Slight inner glow or diffused highlight
- Faint shadow for depth
- Slightly stronger than the background so it feels tappable

Progress Arc
- Soft green accent with slightly higher saturation than the circle
- Muted track behind progress if needed
- Rounded stroke caps

Text
- Main count: dark desaturated green
- Header text: muted deep green
- Secondary labels: softer green-gray

Secondary Buttons
- Light, rounded pill buttons
- Lower contrast than main circle
- Soft shadow only
- Border optional but subtle

Typography
----------
Header Date
- Medium weight
- Small to medium size
- Calm and elegant

Main Count
- Large, bold, highly legible
- Centered inside the circle
- The single strongest text on the screen

Goal Label
- Small to medium size
- Medium weight
- Placed near the main circle but not competing with the main count

Button Labels
- Medium weight
- Clear and readable
- Slightly smaller than header text

Screen Layout Specification
---------------------------
Screen Name
- Home Screen

Overall Hierarchy
-----------------
1. Date header
2. Main circular tap counter
3. Goal progress label
4. Secondary action row

Safe Area and Spacing
---------------------
- Respect status bar and device safe insets
- Use generous top and side padding
- Keep wide spacing around the main circle
- Ensure action buttons remain easy to tap at the bottom portion

Home Screen Layout
------------------
Top Section
- Centered date text
- Format example: "Today · Tuesday, April 23"

Main Section
- Large circular counter area centered on screen
- Progress arc positioned around the upper-right perimeter
- Count number centered inside the circle
- Circle itself is tappable

Support Section
- Goal label below the circle
- Example: "20 / 108"

Bottom Action Row
- Three pill buttons:
  - Undo
  - Reset
  - History

- Equal or near-equal widths
- Spaced evenly
- Lower emphasis than the central tap target

Component Size Guidance
-----------------------
Header
- Height should remain visually light and not dominate the screen

Main Circle
- Largest object on the screen
- Roughly 55 to 70 percent of screen width depending on device

Count Text
- Large enough to remain legible from a quick glance
- Should dominate the inside of the circle

Progress Arc
- Thin to medium stroke
- Rounded ends
- Partial ring, not full circle

Goal Label
- Small, centered, with enough spacing from both circle and action row

Secondary Buttons
- Touch-friendly minimum height
- Wide enough for icon plus label
- Softer than main action in color and elevation

Interaction Specification
-------------------------
Primary Tap Behavior
- User taps the main circle
- Count increments by 1 immediately
- Progress updates immediately
- Count persists locally immediately
- Optional haptic feedback plays
- Press animation confirms interaction

Primary Tap Visual States
-------------------------
Idle
- Circle shows soft gradient and shadow

Pressed
- Slight scale reduction
- Slight shadow reduction
- Optional brightness shift or glow compression
- Animation duration should feel quick and gentle

Released
- Circle returns smoothly to default scale
- Count updates as the interaction completes

Haptic Feedback
---------------
- Trigger on successful increment
- Keep subtle, not strong
- User can disable in settings if that feature is included

Undo Button Behavior
--------------------
- Reverts the most recent increment
- Button should appear available only when undo is possible
- If unavailable, display lower opacity or disabled state

Undo Button Visual Priority
---------------------------
- Softer contrast than Reset and History
- Clearly secondary to the main circle
- Icon and text should remain readable but understated

Reset Button Behavior
---------------------
- Tap opens confirmation dialog
- No direct reset on first tap

Reset Confirmation Dialog
-------------------------
Title
- Reset today's count?

Message
- This will set today's count back to 0.

Actions
- Cancel
- Reset

Dialog Rules
------------
- Cancel dismisses with no change
- Reset clears today's count and saves immediately
- Dialog should use calm styling, not alarming red unless desired

History Button Behavior
-----------------------
- Opens the history screen
- No destructive behavior
- Standard tap feedback

Progress Label Specification
----------------------------
Purpose
- Clarify current progress toward the daily target

Format
- current / target
- Example: 20 / 108

Placement
- Preferred: centered directly below the main circle
- Alternative: above the action row if spacing works better on smaller devices

Visual Treatment
- Smaller than the count
- Clear but understated
- Do not give it stronger contrast than the count

Progress Arc Behavior
---------------------
- Arc reflects current progress toward target
- Visual progress caps at full arc when target is reached
- Count may continue above target numerically
- If no goal is enabled, the arc may be hidden or replaced with decorative neutral arc

Accessibility and Usability
---------------------------
Touch Targets
- All tappable elements should meet comfortable touch size guidelines
- Main circle should be easy to tap anywhere within its visible area

Legibility
- Count must remain readable in bright conditions
- Text contrast should remain sufficient throughout the UI

One-Hand Use
------------
- Main circle should remain within comfortable thumb reach on larger phones as much as possible
- Bottom action row should be easy to access

State Feedback
--------------
- Every tap must feel acknowledged through at least one of:
  - visual animation
  - number update
  - haptic response

Compose Architecture Overview
-----------------------------
Recommended Screen Structure
- HomeScreen
  - HomeHeader
  - CounterTapCircle
  - GoalProgressLabel
  - SecondaryActionRow
  - ResetConfirmationDialog

Compose Component Breakdown
---------------------------

1. HomeScreen
Purpose
- Main container composable for the screen

Responsibilities
- Collect UI state from ViewModel
- Lay out all screen sections
- Pass event callbacks to child composables
- Show reset dialog when needed

Suggested Inputs
- uiState: HomeUiState
- onTapCounter: () -> Unit
- onUndo: () -> Unit
- onResetClick: () -> Unit
- onResetConfirm: () -> Unit
- onResetDismiss: () -> Unit
- onHistoryClick: () -> Unit

2. HomeHeader
Purpose
- Display the date title at the top of the screen

Responsibilities
- Render "Today · Tuesday, April 23" or equivalent formatted date

Suggested Inputs
- title: String

3. CounterTapCircle
Purpose
- Main primary interactive counter component

Responsibilities
- Render large circular background
- Render current count
- Render progress arc
- Handle tap interaction
- Show pressed-state animation
- Trigger haptic feedback through callback or side effect pattern

Suggested Inputs
- count: Int
- progress: Float
- targetEnabled: Boolean
- onTap: () -> Unit
- enabled: Boolean = true

Visual Notes
- Entire circle is clickable
- Arc sits around the outer edge
- Count remains centered
- Use animation APIs for press feedback

Possible Compose APIs
- Box
- Canvas
- animateFloatAsState
- pointerInput or clickable / combinedClickable
- graphicsLayer
- remember
- MutableInteractionSource

4. GoalProgressLabel
Purpose
- Display textual goal progress

Responsibilities
- Render current count and target
- Hide itself if target is disabled, if desired

Suggested Inputs
- count: Int
- target: Int
- visible: Boolean

Output Example
- "20 / 108"

5. SecondaryActionRow
Purpose
- Hold the three lower-priority action buttons

Responsibilities
- Layout Undo, Reset, and History horizontally
- Balance spacing and visual weight
- Handle enabled / disabled states

Suggested Inputs
- canUndo: Boolean
- onUndo: () -> Unit
- onReset: () -> Unit
- onHistory: () -> Unit

6. SecondaryActionButton
Purpose
- Reusable pill-shaped action button

Responsibilities
- Render icon + label
- Support enabled and disabled states
- Remain visually secondary

Suggested Inputs
- label: String
- icon: ImageVector
- enabled: Boolean = true
- onClick: () -> Unit

Visual Notes
- Rounded pill shape
- Soft elevation
- Lower contrast than main circle

7. ResetConfirmationDialog
Purpose
- Confirm destructive reset action

Responsibilities
- Prevent accidental reset
- Present clear action choices

Suggested Inputs
- visible: Boolean
- onConfirm: () -> Unit
- onDismiss: () -> Unit

Possible Compose APIs
- AlertDialog or BasicAlertDialog

Suggested HomeUiState
---------------------
HomeUiState
- formattedDate: String
- count: Int
- target: Int
- targetEnabled: Boolean
- progress: Float
- canUndo: Boolean
- showResetDialog: Boolean

Suggested Event Model
---------------------
User Actions
- TapCounter
- UndoTap
- ResetTap
- ConfirmReset
- DismissResetDialog
- OpenHistory

Compose Layout Outline
----------------------
Suggested screen order:

1. Column root
2. Spacer / top padding
3. HomeHeader
4. Spacer
5. CounterTapCircle
6. Spacer
7. GoalProgressLabel
8. Spacer
9. SecondaryActionRow
10. Optional bottom spacer

Animation Notes
---------------
Counter Press Animation
- Scale to about 0.97 to 0.99 on press
- Return smoothly on release
- Keep animation subtle and quick

Count Change Animation
- Optional slight number transition
- Avoid flashy animations
- The app should feel calm, not gamified

Progress Arc Animation
- Animate arc sweep smoothly when count changes
- Use soft easing

Shadow / Glow Animation
- Optional slight change in shadow during press
- Keep changes restrained

Recommended Compose Libraries / APIs
------------------------------------
Foundation
- Box
- Column
- Row
- Spacer
- Canvas

Material 3
- Surface
- Text
- AlertDialog
- Icon
- IconButton if needed

Animation
- animateFloatAsState
- updateTransition
- AnimatedContent (optional, used carefully)

Input / Feedback
- clickable
- MutableInteractionSource
- LocalHapticFeedback

Navigation Integration
----------------------
Home screen should navigate to:
- History screen from History button
- Settings later if added through top action or overflow menu

State and ViewModel Responsibilities
------------------------------------
ViewModel should handle:
- current count
- target state
- progress calculation
- undo availability
- reset dialog visibility
- tap, undo, and reset events
- local persistence coordination

Compose layer should handle:
- rendering
- animations
- local interaction visuals
- forwarding events upward

Design Constraints
------------------
- Do not reintroduce a separate +1 button inside the circle
- Do not let secondary buttons visually overpower the main circle
- Do not overuse bright contrast or harsh shadows
- Do not add unnecessary navigation clutter to the home screen
- Do not use loud achievement-style animations

Acceptance Criteria for UI
--------------------------
- The main circle is clearly perceived as the tap target
- The main count is the strongest visual element
- Goal text is visible and improves clarity
- Undo appears visually softer than the primary tap target
- Reset requires confirmation
- The screen feels minimal and calm
- The Compose structure is modular and reusable
- The design can scale across common Android phone sizes

Recommended Next Development Task
---------------------------------
Implement the Home screen in Jetpack Compose using:
- HomeScreen
- CounterTapCircle
- GoalProgressLabel
- SecondaryActionRow
- ResetConfirmationDialog

Then connect it to:
- a HomeViewModel
- local persistence layer
- daily target logic
- undo and reset actions

End of Document
